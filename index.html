Routage Inter-VLAN (Projet MLIF)

Contexte : Déploiement d'une architecture "Router-on-a-Stick" pour assurer le routage inter-VLAN et pallier l'absence de lien physique direct vers un serveur Proxmox.

Matériel : Commutateur Allied Telesis AT-9000/28, Hyperviseur local (VirtualBox), PC client physique.

Étapes de réalisation :

  • Configuration Switch : Création des VLANs 50 (Usagers) et 100 (Cœur). Paramétrage du port Client en mode Access et du port Routeur en mode Trunk (802.1Q).
  • Configuration Hyperviseur : Réglage de VirtualBox en "Accès par pont". Modification du registre Windows (MonitorModeEnabled=1) pour forcer la carte réseau à conserver les tags VLAN entrants.
  • Routeur Linux (Debian) : Création de sous-interfaces virtuelles (enp0s3.50 et enp0s3.100) agissant comme passerelles pour chaque réseau (192.168.50.1 et 192.168.100.1).
  • Routage IP : Activation du transfert de paquets au niveau du noyau Linux (net.ipv4.ip_forward=1).
  • Validation : Tests ICMP (Ping) validés entre les VLANs. Analyse des trames taguées via Wireshark sur le lien Trunk.
